The primary entrypoint package for Compiled that provides familiar [React](https://reactjs.org/) APIs to style your app.

```bash
npm install @compiled/react
```

## CSS prop

Use [CSS prop](/api-css-prop) to tie styles to an element.

```jsx
/** @jsxImportSource @compiled/react */

// Object styles
<div
  css={{
    fontSize: 12,
    ...(isPrimary && {
      color: 'blue',
    }),
  }}
/>;

// Template literal styles
<div
  css={`
    font-size: 12px;
  `}
/>;

// Array styles
<div css={[`font-size: 12px;`, isPrimary && { color: 'blue' }]} />;
```

## styled

Use [styled](/api-styled) to create a component that ties styles to an element which comes with built-in behavior such as `ref` and `as` prop support.

```jsx
import { styled } from '@compiled/react';

// Template literal styles
styled.div`
  font-size: 12px;
  color: ${(props) => props.color};
`;

// Object styles
styled.div({
  fontSize: 12,
  color: (props) => props.color,
});

// Array styles
styled.div([
  `
  font-size: 12px;
`,
  { color: (props) => props.color },
]);

// Multi argument styles
styled.div(
  `
  font-size: 12px;
`,
  { color: (props) => props.color }
);
```

## ClassNames

Use [ClassNames](/api-class-names) to use a component where styles are not necessarily tied to an element.

```jsx
import { ClassNames } from '@compiled/react';

// Object styles
<ClassNames>
  {({ css, style }) =>
    children({
      style,
      className: css({ fontSize: 12 }),
    })
  }
</ClassNames>;

// Template literal styles
<ClassNames>
  {({ css, style }) =>
    children({
      style,
      className: css`
        font-size: 12px;
      `,
    })
  }
</ClassNames>;

// Array styles
<ClassNames>
  {({ css, style }) =>
    children({
      style,
      className: css([`font-size: 12px`, { color: 'blue' }]),
    })
  }
</ClassNames>;
```

## CSS

Use [`css`](/api-css) to create styles that can be re-used between components.

```jsx
import { css } from '@compiled/react';

const blackText = css`
  color: black;
`;
```

## Runtime

Functionality used at runtime,
such as style reconciliation and atomic class names compositon.

### CC

Ensures the minimum amount of styles are rendered on the server for child `CS` components.

```jsx
import { CC } from '@compiled/react/runtime';

<CC></CC>;
```

### CS

Moves child CSS to the head of the document in web browsers,
and renders an inline `style` element on the server.

```jsx
import { CS } from '@compiled/react/runtime';

<CS>{['.foo{color:#ff5630;}']}</CS>;
```

### ax

Concats class names together to ensure uniqueness of atomic groups.
Similar in functionality to the `cx` function from the [`classnames`](https://github.com/JedWatson/classnames) package,
optimized for atomic groups.

```tsx
import { ax } from '@compiled/react/runtime';

ax(['_k48pni7l _syaz18rw', '_syazff21']);
// "_k48pni7l _syazff21"
```

### ix

Handles interpolations that may resolve to a falsy value.
When called with a falsy value an empty CSS variable is returned.

```js
import { ix } from '@compiled/react/runtime';

ix(true && '#36B37E'); // "#36B37E"
ix(false && '#36B37E'); // "var(--, )"
```
