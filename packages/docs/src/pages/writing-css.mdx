# Writing CSS

Write CSS using template literals or objects with any of the APIs available with Compiled.

```jsx
import { styled } from '@compiled/react';

const Button = styled.button`
  font-size: 10px;
  font-weight: 500;
  border-radius: 3px;
  border: 1px solid blue;
`;
```

## Features

Compiled has extra features above standard [CSS](https://developer.mozilla.org/en-US/docs/Web/CSS).

### Nested rules

Nest CSS rules within each other.

```jsx
<div
  css={{
    margin: '0 auto',
    div: {
      color: 'red',
      fontSize: 12,
      a: {
        textDecoration: 'none',
      },
    },
  }}>
  <div>
    <a />
  </div>
</div>
```

Use nested rules with caution.
One of the promises of [atomic CSS](/atomic-css) is reducing the total amount of defined styles,
however you will end up creating more selectors when nested.

To ensure minimal creation of CSS rules assign styles directly to the elements instead.

```jsx
<div css={{ margin: '0 auto' }}>
  <div css={{ color: 'red', fontSize: 12 }}>
    <a css={{ textDecoration: 'none' }} />
  </div>
</div>
```

### Nesting selector

The `&` character references the parent selector(s).
In the example below the two selectors (textarea and input) both alter the color on the `:hover` pseudo selector.

```jsx
<div
  css={{
    'textarea,input': {
      color: '#a7a7a7',
      '&:hover': {
        color: '#000',
      },
    },
  }}
/>
```

Becomes:

```css
textarea,
input {
  color: #a7a7a7;
}
textarea:hover,
input:hover {
  color: #000;
}
```

Note how both `textarea` and `input` both have a hover selector.

### Dangling pseudos

These rules are transformed to recieve a nesting selector.

```jsx
<div
  css={{
    ':before': {
      content: 'Hello',
    },
  }}
/>
```

Becomes:

```css
&:before {
  content: 'Hello';
}
```

### Vendor prefixing

Prefixing CSS declarations enables us to ignore slight browser differences and worry instead for experiences we're developing.

```css
user-select: none;
```

Becomes:

```css
-webkit-user-select: none;
-moz-user-select: none;
-ms-user-select: none;
user-select: none;
```

Interestingly not much vendor prefixing is needed today.
